### PHASE 4: User Requests Ride

```
When user taps "Find Driver":
1. Trip request created in Firebase
2. Nearby drivers found via GeoFire
3. Drivers recommended by:
   - Distance (closer = better)
   - Rating (higher = better)
   - Experience (more trips = better)
4. Notification sent to best driver
5. Show "Searching for drivers..." loading screen
6. Timeout after 40 seconds → Try next driver
```

**Code Flow:**
```dart
// User taps "Find Driver"
ElevatedButton(
  onPressed: () {
    // 1. Get available drivers
    setState(() {
      availableNearbyOnlineDriversList =
          ManageDriversMethods.nearbyOnlineDriversList;
    });

    // 2. Create trip request
    _makeTripRequest();

    // 3. Search for driver
    _searchForDriver();

    // 4. Show searching UI
    tripProvider.setSearching(true);
  },
  child: const Text('Find Driver'),
)

// _searchForDriver() implementation
void _searchForDriver() {
  if (availableNearbyOnlineDriversList == null ||
      availableNearbyOnlineDriversList!.isEmpty) {
    // No drivers available
    _showNoDriversDialog();
    return;
  }

  // Get best driver using recommendation service
  List<OnlineNearbyDrivers> recommendedDrivers =
      DriverRecommendationService.recommendDrivers(
    availableNearbyOnlineDriversList!,
    userLat: currentPositionOfUser!.latitude,
    userLng: currentPositionOfUser!.longitude,
  );

  OnlineNearbyDrivers selectedDriver = recommendedDrivers[0];

  // Send notification to driver
  _sendNotificationToDriver(selectedDriver);

  // Remove from available list
  availableNearbyOnlineDriversList!.removeAt(0);
}
```

### PHASE 5: Driver Accepts (Real-time Update)

```
When driver accepts:
1. Firebase listener updates tripStatus → "accepted"
2. TripProvider updates trip state
3. UI changes to show driver details
4. Map updates with driver's real-time location
5. Display:
   - Driver photo
   - Driver name & car details
   - "Call Driver" button
   - Trip status: "Driver is on the way - 5 mins"
```

**Component Usage:**
```dart
TripStatusWidget(
  phase: TripPhase.driverAccepted,
  driverName: 'Ahmed Hassan',
  driverPhone: '0912345678',
  driverPhoto: 'https://...',
  carDetails: 'Toyota Corolla - AA 123',
  onCallDriver: () {
    launchUrl(Uri.parse('tel://0912345678'));
  },
)
```

### PHASE 6: Driver Arrives at Pickup

```
When driver arrives:
1. Firebase listener updates tripStatus → "arrived"
2. TripProvider updates trip state
3. TripStatusWidget changes to show "Driver has arrived"
4. Can call driver or cancel
```

### PHASE 7: Trip in Progress

```
When trip starts:
1. Firebase listener updates tripStatus → "ontrip"
2. Driver's location updates in real-time
3. Route polyline shown on map
4. Trip status shows time to destination
```

### PHASE 8: Trip Ends & Payment

```
When driver completes trip:
1. Firebase listener updates tripStatus → "ended"
2. Trip fare finalized
3. Payment dialog shown
4. User selects payment option
5. After payment:
   - Trip marked as completed
   - Driver rating screen shown
   - App resets to initial state
```

## 📞 TROUBLESHOOTING

### Build fails
```bash
flutter clean
flutter pub get
flutter pub upgrade
flutter run
```

### GeoFire not working
- Verify Firebase Realtime Database is enabled
- Check "onlineDrivers" path exists in database
- Ensure driver app updates location regularly

### Map not showing
- Check Google Maps API key is valid
- Verify API key has Maps SDK enabled
- Check location permissions granted

### Notifications not received
- Verify Firebase Cloud Messaging is setup
- Check device token is correctly stored
- Verify notification payload format
